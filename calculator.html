EECU Calculator

The goal of this project was to create and code a design for a monthly budget calculator.

Languages HTML, CSS, and JavaScript.

Tools

Figma to create a high fidelity design, Visual Studio Code for development and Git/GitHub to enable collaboration with partners.

Concepts

DOM Manipulation: I expanded my knowledge in regards to handling multistep pages and the basic concepts of their logic. Communication: I, as a Developer, enhanced my ability to listen to directions in a way that was efficient and followed what my partner wanted. In the role of Project Manager, I refined how I delivered instructions to grant more freedom in the actual execution without disregarding the original point of view for the project.

Challenges & Solutions

The Problem: I had a hard time wrapping my head around the logic required for more complex features, such as the piechart. The Solution: I watched as others would develop on my project and see how they would deal with the logic surrounding the chart, asking questions when they arose. Later in the project when we were able to alter our own code I then went through and had to mend some issues the graph had providing me with experience gathering and plugging in data to the graph. The Problem: As with any project there were many bugs throughout the coding process. However, I felt it was taking me a considerable amount of time to find and squash them. The Solution: When issues arose I had the opportunity to observe the different problem solving strategies they implemented. Taking each of these ideas and combining them together allowed me to quickly identify and begin solving these problems faster than before. The Problem: Due to there being many hands and minds on this project it was sometimes difficult to understand certain portions of the code and what it did. The Solution: To counteract this I would consistently attempt to understand the logic as it was written to both increase my own knowledge for later use, but to also explain the code to further developers if they too encountered issues wrapping their brains around it.

Habits of Mind

Persistence: When my CSS animations (using @keyframes) weren’t firing correctly across different browsers, I spent three days debugging the rendering engine quirks rather than simplifying the design. Communication: I consistently acknowledged the skill of peers gaining more tools to be implemented later on by asking questions and actively participating in the logical thinking processes for the code they were doing.
